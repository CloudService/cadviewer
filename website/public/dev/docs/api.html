2.2       Retrieve the file contents

It is used to get all the immediate children folders and files. It doesn't get the indirect children recursively. It is called by Browser.

 

Request

 

GET /api/1.0/files/:id

 

Response

The response is the full file object.

 

3         Tasks

    "type": "task",

    "id": "358034830",

    "action": "import",

    "file_id": "2342342",

    "model_id": "345345435"

}

 

3.2       Import a file

It is used to summit a job to import a file. It is called by Browser.

 

Request

POST /api/1.0/tasks/import

 

{

    "type": "file",

    "id": "128420334"

}

 

Response

The response is the mini model object as below

 

{

    "type": "model",

    "id": "358034830"

}

 

3.3       Get a pending task

It is used to get all the pending jobs. It is called by the worker.

The pending task will be deleted by the sever after this API is called.

 

Request

GET /api/1.0/tasks/0

 

Response

Return a pending task

4.2       Get model object

It is used to get the mesh data of an imported model. It is called by Browser.

The model will be deleted by the server after this api is called.

 

Request

GET /api/1.0/models/:id

 

Response

The response is the full model object

 

Upload the mesh

It is used to upload the mesh to the server. It is called by worker.

 

Request

PUT /api/1.0/models/:id

 

{

     "mesh":"...tbd..."

}

 

Response

The response is the mini model object.
